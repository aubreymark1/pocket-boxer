# UX Audio Battle Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复当前版本中首页按钮特效、出拳测试布局、出拳音效、人机对战转场速度与重叠问题，让演示版在手机端更稳定、更清晰、更有冲击感。

**Architecture:** 这次优化不改页面主流程，只做表现层和节奏层打磨。核心思路是把问题拆成 3 个层面处理：按钮与倒计时等通用动效、出拳页与结果页的布局约束、战斗页的回合动画与节奏控制。所有改动优先落在现有的 `index.html`、`styles.css`、`src/app.js`、`src/feedback.js` 上，避免扩散到不必要的重构。

**Tech Stack:** HTML, CSS, JavaScript, Web Audio API, Vibration API

---

## Files To Touch

- Modify: `index.html`
  - 调整出拳页、战斗页的 DOM 结构锚点，给标题、沙包、战斗状态区增加更稳定的布局容器。
- Modify: `styles.css`
  - 修复首页按钮 `PUNCH!` 特效定位与停留时长。
  - 收缩倒计时 `PUNCH!` 的字号、描边和漫画感样式。
  - 给出拳页和战斗页增加防重叠布局约束与响应式缩放。
  - 拉长回合状态页的视觉停留节奏。
- Modify: `src/app.js`
  - 调整倒计时 `PUNCH!` 的显示时长。
  - 调整战斗播放时序，放慢页面切换和回合自动推进。
  - 为首页按钮增加更稳定的特效触发点。
  - 检查出拳阶段和战斗阶段文本更新时机，避免和主体视觉同时抢位。
- Modify: `src/feedback.js`
  - 补强“出拳爆炸音效”，让当前 `punch()` 更接近漫画式击打爆点。
  - 视需要拆分出 `punchCharge()` / `punchImpact()`，避免倒计时音和击中音混在一起。
- Verify: `src/battle.js`
  - 不作为本轮主改文件，但需要确认战斗阶段数据字段足够支持更慢的回放节奏。

## Task 1: 首页按钮 `PUNCH!` 弹字修正

**目标**

- 让首页按钮按下时的 `PUNCH!` 弹字始终贴近按钮本体出现。
- 停留时间更长，用户能清楚看到，而不是一闪而过。
- 保留当前霓虹街机风格，不改成完全另一种视觉语言。

- [ ] 检查 `styles.css` 中 `.accent-button:active::after` 和 `@keyframes punchPop` 的当前位置、缩放和透明度曲线。
- [ ] 把弹字锚点从“按钮中心上方”改成“按钮右上或按钮正上更近的位置”，减少飘离感。
- [ ] 把动画总时长从 `0.3s` 提升到 `0.45s ~ 0.6s`，并让 `opacity: 1` 的区间更长。
- [ ] 给按钮容器补充 `overflow: visible` 的检查，避免文字被按钮或父容器裁掉。
- [ ] 手动验证首页三个主按钮，确认只有主 CTA 使用该特效且位置稳定。

**验收标准**

- `PUNCH!` 文字出现在按钮附近，不悬空偏移。
- 连续点击时不会闪到屏幕边缘。
- 停留时间足够看清，但不会拖慢按钮响应。

## Task 2: 倒计时 `PUNCH!` 视觉与节奏修正

**目标**

- 解决倒计时页 `PUNCH!` 字样过大、超屏、缺少漫画感的问题。
- 保留现在“数字倒数 + 最后一击提示”的核心节奏。

- [ ] 检查 `index.html` 中 `#countdown-display` 的内联字号设置，以及 `src/app.js` 中倒计时结束后 `PUNCH!` 的展示时长。
- [ ] 将 `#countdown-display` 的 `PUNCH!` 状态与数字状态分开控制：
  - 数字 `3/2/1` 保持大号；
  - `PUNCH!` 使用更小字号、倾斜、描边、阴影和爆点感。
- [ ] 在 `styles.css` 中新增 `#countdown-display.is-punch-cue` 样式，避免直接复用数字字号。
- [ ] 将 `src/app.js` 中 `await new Promise((resolve) => setTimeout(resolve, 450));` 调整为更合理的 `600ms ~ 800ms` 区间，给用户足够的视觉确认时间。
- [ ] 手动验证窄屏手机尺寸下 `PUNCH!` 不会超出左右边界。

**验收标准**

- `PUNCH!` 不超屏。
- 更像漫画冲击字，而不是简单放大文字。
- 出现后用户能明确知道“现在该出拳了”。

## Task 3: 出拳页与结果页的防重叠布局修复

**目标**

- 解决“挥动手机，击打沙包！”与沙包主体重叠的问题。
- 解决战斗回合内文字与主体视觉抢占同一竖屏空间的问题。

- [ ] 检查 `index.html` 中 `screen-punch`、`screen-result`、`screen-battle-status` 的结构，梳理当前标题、沙包、信息卡、按钮的先后顺序。
- [ ] 在 `styles.css` 中为 `screen-punch-layout`、`screen-result-layout`、`screen-battle-layout` 建立稳定的垂直分区：
  - 顶部：标题 / HUD
  - 中部：主体视觉（沙包或对战舞台）
  - 底部：说明文案 / 按钮
- [ ] 给 `punch-stage`、`result-stage`、`battle-arena` 增加 `min-height`、`max-height` 和 `flex` 约束，避免主体元素挤压文字区。
- [ ] 收紧 `#punch-instruction`、`#battle-status-title`、`#battle-status-copy` 的字号和行高，并给长文案增加最大行数意识。
- [ ] 检查 `transform: scale(1.2)` 等放大值，必要时在小屏幕上降到 `1.0 ~ 1.08`，优先保证不重叠。
- [ ] 手动验证首页、出拳页、结果页、战斗状态页在常见手机竖屏下不再出现标题压沙包、文案压舞台的问题。

**验收标准**

- 出拳页标题、沙包、按钮三者不会重叠。
- 战斗状态页标题、对战舞台、状态文案三者层次清晰。
- 不需要靠极端缩小字体来换取布局稳定。

## Task 4: 出拳爆炸音效增强

**目标**

- 在当前已有音频模块基础上，让出拳和命中更像“漫画爆点 + 街机拳击机”的声音。
- 保持轻量，不引入外部音频资源依赖。

- [ ] 审查 `src/feedback.js` 中现有 `punch()` 与 `hit(score)`，确认“出拳动作音”和“命中爆点音”目前的分工。
- [ ] 将 `punch()` 从单纯下坠音改成“短促空气切割 + 低频爆点”的组合。
- [ ] 如有必要，新增 `punchImpact()` 或增强 `hit(score)`，让手机挥动捕获到击中阈值时能明显听到“砰”的感觉。
- [ ] 检查 `src/app.js` 中 `motionPunchCuePlayed` 与 `delta > 5` 的触发时机，避免过早播放，尽量贴近真实命中瞬间。
- [ ] 手动验证：
  - 倒计时音、出拳音、命中音三者不会互相盖住。
  - 连续快速测试时声音不会失控叠加成噪音。

**验收标准**

- 挥拳命中时有明显“爆炸 / 爆点”感。
- 声音比现在更有力量感，但不会刺耳。
- 不依赖额外音频文件也能稳定演示。

## Task 5: 人机对战转场放慢与回合节奏优化

**目标**

- 解决人机对战“页面之间切得太快”的问题。
- 让每一回合玩家能看清：自己出拳、对方受击、对方反击、回合结算。

- [ ] 检查 `src/app.js` 中 `playBattleStatusAnimation()` 与 `showBattleStatus()` 的各段 `wait()` 时长。
- [ ] 将当前 `180 / 560 / 180 / 620 / 700` 这类节奏参数提取成具名常量，统一调节。
- [ ] 放慢以下阶段：
  - 玩家出拳前摇
  - 命中停顿
  - 机器人反击前摇
  - 回合结束后的“进入下一回合”停顿
- [ ] 给“自动进入下一回合”增加更清晰的可感知缓冲，必要时只在非最终回合保留自动推进，并保证按钮文案同步。
- [ ] 检查战斗状态页的标题和描述文案刷新顺序，避免在动画进行时立刻替换全部文字，造成视觉拥挤。
- [ ] 手动验证 3 回合完整流程，确认每个阶段都能被看清，不像闪屏。

**验收标准**

- 玩家能看清每一回合发生了什么。
- 自动推进仍然流畅，但不仓促。
- 最终结果页前有明确的收束感。

## Task 6: 联调与回归验证

**目标**

- 确保这轮表现层优化没有打断主流程。

- [ ] 使用本地预览地址手动验证 `首页 -> 安全页 -> 倒计时 -> 出拳 -> 结果 -> 人机对战 -> 三回合 -> 结算` 全链路。
- [ ] 在手机窄屏和桌面开发者模式下各跑一遍，重点观察：
  - 首页按钮弹字位置
  - `PUNCH!` 倒计时冲击字
  - 出拳页标题和沙包间距
  - 战斗页标题和舞台间距
  - 音效触发是否符合预期
- [ ] 对最近修改的 `index.html`、`styles.css`、`src/app.js`、`src/feedback.js` 运行诊断检查，及时修复语法或引用错误。
- [ ] 只在以上问题全部通过后，再进入下一轮“素材接入 + 1000 分体系 + 结果滚分”优化。

## Suggested Execution Order

- 先做 Task 1 和 Task 2，快速解决最明显的首页与倒计时问题。
- 再做 Task 3，统一修复出拳页和战斗页重叠。
- 然后做 Task 4，补强出拳爆炸音效。
- 最后做 Task 5，调慢对战节奏并做 Task 6 联调。

## Risks

- `index.html` 中存在大量内联样式，若只改单点数值容易反复回归，建议优先抽到 `styles.css` 中集中管理。
- `battle` 流程当前是自动推进型设计，放慢后要注意不要让用户误以为页面卡住。
- Web Audio 在移动端可能受首次交互解锁限制，联调时需确保先发生过一次用户点击。

## Done Definition

- 首页按钮弹字位置正确、可见时长合理。
- 倒计时 `PUNCH!` 不超屏且更有漫画感。
- 出拳页和战斗页的文字不再压住沙包或主视觉。
- 出拳命中时有更明确的爆炸音效。
- 人机对战每一回合的切换节奏明显变慢且更清晰。
